**Known issue: "full" stage intermittent crash (~70% failure rate)**

The end-to-end "full" stage crashes with "Illegal barrier arrive operation". Root cause identified: **semaphore array overflow**.

With hidden_dim=512 and NUM_CONSUMER_WARPS=16, `matvec_pipeline::find_stage_pages()` returns STAGE_PAGES=1, giving INPUT_PIPELINE_STAGES=12. This inflates semaphore counts:

```
matvec_pipeline::SEM_COUNT       = 1 + (12 + 3) * 2 = 31
rms_matvec_pipeline              = 31 + 1            = 32  (rms_scale_arrived)
QKV (rms_qkv_rope_append)        = 32 + 1            = 33  (rope_arrived)       ← OVERFLOW
RmsRouterUpgate                   = 31 + 2            = 33  (rms_weights + router_done) ← OVERFLOW
```

`DYNAMIC_SEMAPHORES = 32` in `include/config.cuh`. The 33rd semaphore (`semaphores()[32]`) writes out of bounds into `scratch[0..1]` in the `instruction_state_t` struct (`include/util.cuh:11-19`). This corrupts mbarrier state. The controller's invalidation loop also reads out of bounds (`laneid < num_semaphores` where `num_semaphores=33`), corrupting scratch memory for the next instruction in the same ring slot.

**Why individual stages pass:** No ring-slot reuse — the controller never invalidates a prior instruction's semaphores.

**Fix options (not yet implemented):**
1. Cap `INPUT_PIPELINE_STAGES` in `matvec_pipeline.cuh:36-37` to keep total SEM_COUNT <= 32
2. Increase `DYNAMIC_SEMAPHORES` to 64 in `config.cuh:22`
3. Add `static_assert(SEM_COUNT <= Config::DYNAMIC_SEMAPHORES)` in each op's `init_semaphores`

The oproj failure is in `matvec_adds.cu` (output projection + residual add). `hidden_states` values are completely wrong (max_adiff=771, values in opposite directions). This is the same matvec_pipeline code used by the old 3-opcode MoE — it's not a regression from our fusion work.

The semaphore overflow does NOT exist at full scale (INPUT_PIPELINE_STAGES=1, SEM_COUNT~14).

## How to Continue

### Priority 1: Fix the OProj Full-Scale Bug

This blocks all full-scale testing. The issue is in `demos/mixtral/matvec_adds.cu`, which implements the o_proj op using `matvec_pipeline`.

**Debugging approach:**
1. Run `uv run python demos/mixtral/debug_mixtral_mk_full.py --stage oproj` to reproduce
2. The oproj uses a reduction-based matvec (hidden_states = o_proj_weights @ attn_out + residual). The `store_add_async` writes the result to hidden_states
3. Check if the issue is in the matvec computation itself or in the residual accumulation
4. Compare against the PyVM reference in `megakernels/mixtral/python_vm.py:solve_oproj`
5. The `matvec_adds.cu` store function may have addressing issues at full scale — check the TMA coordinates and weight depth indexing

**Key files:**
- `demos/mixtral/matvec_adds.cu` — the oproj CUDA kernel
- `demos/mixtral/matvec_pipeline.cuh` — the generic pipeline (shared with all matvec ops)
- `megakernels/mixtral/python_vm.py:solve_oproj` — PyVM reference
- `megakernels/demos/latency/python_vm.py:matvec_with_residual` — the helper it calls

### Priority 2: Fix the SMALL_TEST Semaphore Overflow

This is needed if you want to use the SMALL_TEST build for debugging. Quick fix:

In `demos/mixtral/matvec_pipeline.cuh`, cap INPUT_PIPELINE_STAGES:
```cpp
static constexpr int INPUT_PIPELINE_STAGES =
    min((Config::NUM_PAGES - 1) / STAGE_PAGES, 6);
```

This limits IPS to 6, giving base SEM_COUNT = 1 + (6+3)*2 = 19, leaving room for extra semaphores.

Add static_asserts in `rms_matvec_rope_append.cu` and `rms_router_matvec.cu`:
```cpp
static_assert(pipeline::SEM_COUNT + N <= Config::DYNAMIC_SEMAPHORES);
```

### Priority 3: Full-Scale End-to-End Validation

Once oproj is fixed, run:
```bash
uv run python demos/mixtral/debug_mixtral_mk_full.py --no-stop
```

Warning: the PyVM reference is extremely slow at full scale (~30min+ for all stages) because it does 4096x4096 matvecs in Python. For faster iteration, test individual stages with `--stage <name>`.

To stress-test the full pipeline (no semaphore overflow at full scale):
```bash
for i in $(seq 10); do
  uv run python demos/mixtral/debug_mixtral_mk_full.py --stage full 2>&1 | tail -1
done
```
